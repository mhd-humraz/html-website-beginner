Sectioning Elements

What are Semantic Sectioning Elements?

Semantic HTML5 sectioning elements provide meaning to both browsers and developers about the structure of content.

Key Sectioning Elements:

  • <header> - Introductory content or navigation
  • <nav> - Navigation links
  • <main> - Dominant content of the document
  • <article> - Self-contained composition
  • <section> - Thematic grouping of content
  • <aside> - Content indirectly related to main content
  • <footer> - Footer for its nearest ancestor

Best Practice:

Use <main> only once per page, as it represents the primary content.

Example Structure

<body>
    <header>
        <h1>Page Title</h1>
        <nav>...</nav>
    </header>
    
    <main>
        <article>
            <h2>Article Title</h2>
            <section>
                <h3>Section Heading</h3>
                <p>Content...</p>
            </section>
        </article>
        
        <aside>
            <h2>Related Info</h2>
            <p>Side content...</p>
        </aside>
    </main>
    
    <footer>
        <p>Copyright info</p>
    </footer>
</body>

Accessibility

Why Semantic HTML Matters for Accessibility

Semantic HTML provides crucial information to assistive technologies like screen readers, helping users with disabilities navigate and understand your content.

Key Accessibility Features:

  • Landmark roles: Sectioning elements automatically create landmarks that screen readers can navigate
  • Heading hierarchy: Proper use of <h1> to <h6> creates a document outline
  • Alternative text: alt attributes for images
  • ARIA attributes: Additional accessibility information when needed

Common Mistake:

Avoid using <div> for everything. Semantic elements provide built-in accessibility features that divs lack.

Accessibility Checklist

  1. Use proper heading structure (h1-h6 in order)
  2. Include alt text for meaningful images
  3. Ensure all interactive elements are keyboard accessible
  4. Use semantic elements instead of divs when appropriate
  5. Provide text alternatives for non-text content
  6. Ensure sufficient color contrast
  7. Use ARIA landmarks when needed

ARIA Landmark Example

<nav aria-label="Main navigation">
    <ul>
        <li><a href="#home">Home</a></li>
        <li><a href="#about">About</a></li>
    </ul>
</nav>

SEO Basics

How Semantic HTML Helps SEO

Search engines use semantic markup to better understand your content's structure and meaning, which can improve your rankings.

SEO Benefits of Semantic HTML:

  • Clear content hierarchy helps search engines understand your page
  • Proper heading structure indicates content importance
  • Semantic elements can be weighted differently in search algorithms
  • Improved accessibility often correlates with better SEO

Pro Tip:

Content within <main> is considered primary content by search engines, while <aside> content may be weighted less.

Essential SEO Elements

Element SEO Benefit
<title> Most important on-page SEO element
<meta name="description"> Often used as the snippet in search results
<h1>-<h6> Indicates content hierarchy and importance
<img alt="..."> Helps with image search and accessibility
<article> Indicates self-contained, potentially syndicatable content

Structured Data Example

Combine semantic HTML with structured data (like Schema.org) for even better SEO results:

<article itemscope itemtype="http://schema.org/Article">
    <h1 itemprop="headline">Article Title</h1>
    <div itemprop="author" itemscope itemtype="http://schema.org/Person">
        By <span itemprop="name">Author Name</span>
    </div>
    <div itemprop="articleBody">
        <p>Article content...</p>
    </div>
</article>
